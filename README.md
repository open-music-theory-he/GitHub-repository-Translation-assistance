# GitHub Repository Translation Assistance

אוטומציה בת שני שלבים לתרגום קבצי ה-Markdown של המאגר
[`open-music-theory-he/open-music-theory-he.github.io`](https://github.com/open-music-theory-he/open-music-theory-he.github.io)
לעברית, באמצעות Gemini API. הכל רץ כ-GitHub Actions **במאגר הזה בלבד** –
המאגר המקורי נקרא ממנו לקריאה בלבד ולעולם לא נכתב אליו.

## מבנה

```
.github/workflows/
  extract-sentences.yml   # סקריפט 1 - הפעלה חד-פעמית ידנית
  translate.yml           # סקריפט 2 - רץ אוטומטית כל שעתיים
scripts/
  extract_sentences.py
  translate_sentences.py
data/
  sentences.json          # פלט של סקריפט 1 (נוצר בהרצה הראשונה)
  translations.json       # פלט מצטבר של סקריפט 2
requirements.txt
```

## הגדרה חד-פעמית (חובה לפני ההרצה)

1. **Secret**: Settings → Secrets and variables → Actions → New repository
   secret בשם `GEMINI_API_KEY` עם מפתח ה-API של Gemini שלך.
2. **הרשאות כתיבה**: Settings → Actions → General → Workflow permissions →
   בחר "Read and write permissions" ושמור. זה נדרש כדי שה-workflow-ים יוכלו
   ל-commit ול-push את קבצי ה-JSON בחזרה למאגר הזה (הם משתמשים ב-
   `GITHUB_TOKEN` המובנה, אין צורך ב-PAT נפרד).
3. מכיוון שמאגר המקור ציבורי, אין צורך בטוקן כדי לקרוא ממנו. אם בעתיד הוא
   יהפוך לפרטי, יש להוסיף secret בשם `SOURCE_REPO_TOKEN` (PAT עם הרשאת קריאה
   למאגר המקור) ולהסיר את ההערה בשורות המתאימות ב-`extract-sentences.yml`.

> **הערה לגבי האסימונים ששיתפת בשיחה**: כדי לא לשמור סודות בקוד, בחרתי שלא
> להטמיע את ה-PAT של GitHub או את מפתח ה-Gemini בקבצים עצמם – רק להשתמש ב-
> secrets כמתואר למעלה. מומלץ בכל מקרה **לסבב (rotate)** את שני האסימונים
> ששלחת בצ'אט, כדי שלא יישארו תקפים אם מישהו אחר יראה את ההיסטוריה.

## הרצה

1. הפעל ידנית את workflow **"1 - Extract sentences from source repo"** מתוך
   לשונית Actions (Run workflow). הוא יווצר את `data/sentences.json`.
2. workflow **"2 - Translate sentences (Gemini)"** ירוץ אוטומטית כל שעתיים
   (אפשר גם להריץ אותו ידנית לבדיקה). בכל הרצה:
   - אם `data/translations.json` מסומן `completed: true` – הסקריפט יוצא מיד
     ולא עושה כלום.
   - אחרת הוא ממשיך מהמשפט הראשון שעדיין אין לו תרגום.
   - יש השהיה קצרה בין קריאה לקריאה, ו-backoff גדל בין ניסיונות חוזרים על
     אותו משפט (ברירת מחדל: 60 שניות אחרי כישלון ראשון, 300 שניות אחרי כישלון
     שני; כישלון שלישי ברציפות = שמירה ועצירה עד ההרצה הבאה).
   - אחרי 30 דקות מתחילת ההרצה, הסקריפט מסיים את המשפט הנוכחי ועוצר, גם
     בהיעדר שגיאות.

כל ההתנהגות ניתנת לכיוונון דרך משתני הסביבה שמוגדרים (בהערה) בתוך
`translate.yml`: `GEMINI_MODEL`, `TIME_BUDGET_SECONDS`, `SLEEP_BETWEEN_SECONDS`,
`RETRY_DELAYS_SECONDS`.

## בחירת מודל Gemini

בדקתי את תיעוד המחירים העדכני של Google (יולי 2026):

| מודל | קלט / פלט למיליון טוקן | הערה |
|---|---|---|
| `gemini-2.5-flash-lite` | $0.10 / $0.40 | הזול ביותר, מתאים אם עלות היא השיקול המרכזי |
| **`gemini-2.5-flash`** (ברירת מחדל כאן) | $0.30 / $2.50 | איזון טוב בין איכות לעלות עבור תרגום טכני/מוזיקלי עם כללי פורמט מחייבים |
| `gemini-3.5-flash` | $1.50 / $9.00 | איכות/הבנה גבוהה יותר, יקר משמעותית |

⚠️ `gemini-2.0-flash` הוצא משימוש (הופסק ב-1 ביוני 2026) ולכן לא נעשה בו שימוש.

בחרתי ב-`gemini-2.5-flash` כברירת מחדל: המשימה דורשת עמידה מדויקת בכללי
פורמט ומינוח (שמירת Markdown, הבחנה בין סימנים גרפיים למילוליים וכו'), ומודל
Flash Lite עלול להחליק בכללים העדינים האלה על פני נפח גדול של משפטים, בעוד
ש-Flash רגיל נותן שיפור משמעותי באיכות תמורת תוספת עלות מתונה. אפשר לשנות
בקלות ל-`GEMINI_MODEL=gemini-2.5-flash-lite` (לחיסכון) או
`gemini-3.5-flash` (לאיכות מקסימלית) בקובץ ה-workflow.

## הנחות/מגבלות בסקריפט 1

- "משפט" מוגדר כאן כשורה לא-ריקה בקובץ ה-MD (בהתאם לניסוח "יעבור שורה
  אחר שורה" בתיאור סקריפט 2).
- מוסר: כותרות (`#`...), סימון ציטוט (`>`), תבליט רשימה (`-`/`*`/`+`/`1.`
  בתחילת שורה בלבד, פעם אחת), קווי הפרדה אופקיים (`---`/`***`), שורות הפרדה
  של טבלאות (`| --- | --- |`), YAML front-matter של Jekyll, ובלוקי קוד
  מגודרים (` ``` `).
- נשמר ללא שינוי: כל עיצוב שאינו ברמת השורה כולה - `**bold**`, `*italic*`,
  ` `code` `, קישורים `[text](url)` וכו', גם אם במקרה מסוים הם משתרעים על פני
  השורה כולה.
- שורות תוכן של טבלאות (לא שורות ההפרדה) נשמרות כמות שהן, כולל תווי `|`.

אם תרצה התנהגות שונה (לדוגמה: פיצול לפי משפטים דקדוקיים אמיתיים ולא לפי
שורות, או טיפול שונה בטבלאות) – אפשר לעדכן את `scripts/extract_sentences.py`
בהתאם.
